# Tech stack

> **Note:** Earlier drafts of this doc described a Rust + Tauri stack.
> That implementation lives on the `rust-legacy` branch but isn't the
> active code anymore - we pivoted to Go + Wails for opkssh support
> (the russh fork rejects opkssh's `valid_before=u64::MAX` certs).
> What follows describes the current stack.

## Backend (Go 1.26)

| Module | Purpose |
|---|---|
| `github.com/wailsapp/wails/v3` | Desktop shell + IPC (alpha2.111) |
| `modernc.org/sqlite` | SQLite, pure Go (no CGO) |
| `golang.org/x/crypto/ssh` | SSH client |
| `golang.org/x/crypto/ssh/agent` | Agent integration |
| `golang.org/x/crypto/argon2` | Vault KDF (Argon2id, interactive params) |
| `golang.org/x/crypto/chacha20poly1305` | Vault AEAD (XChaCha20-Poly1305) |
| `github.com/zalando/go-keyring` | OS keychain fallback for machine-bound auto-unlock |
| `github.com/openpubkey/openpubkey` | OpenPubkey OIDC client |
| `github.com/openpubkey/opkssh` | opkssh cert issuance + verification (as a Go library, no external binary) |
| `github.com/aymanbagabas/go-pty` | Cross-platform PTY (creak/pty on Unix, ConPTY on Windows) |
| `github.com/google/uuid` | Stable ids for tree rows, sessions, dynamic entries |
| `golang.org/x/net/proxy` | SOCKS5 dialer for the HTTP probe |
| `gopkg.in/yaml.v3` | opkssh provider config + Ansible YAML inventory parser |

The dynamic inventory providers (`internal/inventory/{proxmox,
hetzner,digitalocean,linode,vultr,scaleway,aws_ec2}.go`) hit
provider HTTP APIs directly with `net/http` - no SDK dependencies.
The Ansible provider (`internal/inventory/ansible.go`) parses a
local `.ini` or `.yml` inventory file in-process; no Python
runtime / `ansible-inventory` shell-out.

## Frontend (Node 20)

| Package | Purpose |
|---|---|
| `svelte` 5 | UI framework (runes API: $state, $derived, $effect, $props) |
| `vite` 5 | Build tool, dev server |
| `@wailsio/runtime` | v3 frontend runtime (Events, dialogs, file drop) |
| `@xterm/xterm` 5 | Terminal emulator |
| `@xterm/addon-fit` | Auto-resize |
| `@xterm/addon-webgl` | WebGL renderer |
| `@xterm/addon-web-links` | Clickable URLs |
| `@xterm/addon-search` | Scrollback search |
| `lucide` | Icon set (replaced emoji placeholders) |
| `typescript` 5 | Static types |

Wails autogenerates TS bindings into `frontend/bindings/`. The
`frontend/src/lib/api.ts` facade wraps those with plain-object
interfaces (the autogenerated classes carry a `convertValues`
member that trips strict TS).

## Tooling

- **Task** (`Taskfile.yml`) for build orchestration. Top-level
  routes by OS (`task windows:build`, `task linux:build`); per-OS
  Taskfiles in `build/{os}/`.
- **npm** for the frontend.
- **gofmt + go vet** for Go; `svelte-check` for the frontend.
- **gitlab.com CI** for multi-arch container build of the release
  server (sibling repo).

## CI

GitHub Actions (`.github/workflows/release.yml`): tag push runs
tests, builds linux/windows amd64+arm64 and the signed android APK,
and publishes a GitHub Release. sshtool.app mirrors GitHub Releases;
nothing is uploaded there.

## Version pinning

- Go: 1.26 (see `go.mod`)
- Node: 20.x LTS
- Wails CLI v3: matches the alpha tag pinned in `go.mod` (currently
  alpha2.111)

## What we don't use, and why

- **Electron** - RAM, bundle size, brand reasons.
- **openssh-portable / system ssh as primary** - predictable cross-
  platform behaviour from `x/crypto/ssh`. System ssh wasn't needed.
- **russh / Rust SSH stack** - rejects opkssh "forever" certs
  (`valid_before=u64::MAX`) at the ssh-key parsing level. Rewriting
  the cert breaks the CA signature. `golang.org/x/crypto/ssh`
  accepts it natively.
- **Slint / Iced / egui / native widgets** - building a file
  manager with drag-drop + xterm widget is months. Webview wins on
  UX productivity here.
- **CGO SQLite** - `modernc.org/sqlite` is pure Go, cross-compiles
  trivially. Cross-build from WSL to Windows works because we don't
  touch CGO.
